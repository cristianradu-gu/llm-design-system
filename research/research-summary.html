Design System Tech Stack Research Summary

Research question: "Research the best tech stack for a design system. The system will be used to power a SaaS web application for the energy industry, written in TypeScript. The design system will be used by AIs (Figma Make, Cursor, others) so availability to LLMs is a big plus."

LLM Framework Primitives Styling Tokens Documentation Build Tool
Composer-1 React + TypeScript
Details
  • Industry standard for SaaS applications
  • Strong TypeScript support with excellent type inference
  • Component-based architecture aligns with design system principles
  • Extensive ecosystem and community support
  • LLM-friendly: TypeScript types provide clear contracts for AI tools
 Radix UI + shadcn/ui
Details
  • Radix UI: Unstyled, accessible primitives (headless components)
  • WCAG compliant out of the box
  • Maximum flexibility for custom styling
  • shadcn/ui approach: Copy-paste component pattern
  • Components live in your codebase (not node_modules)
  • LLM-friendly: AI can read and modify component source directly
 Tailwind CSS
Details
  • Explicit class names are more readable by LLMs
  • Utility classes provide clear intent (text-lg, bg-primary-500)
  • Design tokens map directly to Tailwind config
  • Better performance (no runtime CSS-in-JS overhead)
  • Easier for AI to generate correct styling
 W3C Design Tokens Format (JSON)
Details
  • Standard JSON structure that LLMs can easily parse
  • TypeScript types generated from tokens
  • Multi-platform support (CSS variables, iOS, Android)
  • Tools: style-dictionary or tokens-studio
 Storybook
Details
  • Industry standard for design system documentation
  • Interactive component playground
  • LLM-friendly: Stories are TypeScript/JavaScript that AI can understand
  • Auto-generates component API documentation
  • Integrates with design tokens and Figma
 Vite or Next.js
Details
  • Vite: Fast, modern, excellent TypeScript support
  • Next.js: If design system is part of larger application
  • Both support TypeScript and modern ES modules
Gemini 3 Flash React 19 + TypeScript

  • Strong TypeScript support
  • Component-based architecture
 Radix UI
Details

 Tailwind CSS
Details

  • AI models have been trained extensively on Tailwind
  • Much better at generating UI using utility classes than custom CSS-in-JS
  • Semantic training data: LLMs are "fluent" in Tailwind
  • Zero-context styling: styles are in HTML
 Style Dictionary (W3C Design Token Standard)
Details
  • Store design tokens in standard JSON format
  • Can provide JSON file to LLM context

  • Allows AI to "know" brand rules programmatically
 Storybook (MDX) + llms.txt

Strong Consensus Recommendations

  • React + TypeScript: Unanimous choice across all LLMs. Strong LLM training data, type safety, and component-based architecture.
  • Radix UI: Unanimous choice for primitives. Unstyled, accessible, headless components that provide solid foundation.
  • W3C Design Tokens / Style Dictionary: Unanimous choice. Standard JSON format that's LLM-friendly and multi-platform.
  • Storybook: Unanimous choice for documentation. Interactive, LLM-readable, and industry standard.
  • Tailwind CSS: Strong majority (75%) recommend Tailwind for its LLM-friendly utility classes and extensive training data.
  • shadcn/ui pattern: Multiple LLMs recommend copy-paste component pattern for AI accessibility.
  • Model Context Protocol (MCP): Mentioned by several LLMs as key for AI integration with Figma and Cursor.
  • llms.txt: Recommended by multiple LLMs as a way to provide LLM-friendly documentation.
